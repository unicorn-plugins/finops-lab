> **상속**: 본 addendum은 `{DMAP_PLUGIN_DIR}/resources/guides/office/pptx-build-guide.md`를 **상위 표준**으로 함.

## 1. 산출물별 표준 흐름 (MUST)

각 FinOps 산출물은 의사결정·학습 맥락이 다르므로 **산출물 유형별 표준 흐름**을 채택:

| 산출물 | 전달 핵심 | 흐름 | 권장 장수 |
|--------|----------|------|----------|
| **why-statement** | 왜 FinOps인가? | 문제 → 비즈니스 이슈 → 3 가치(절감/가속/신뢰) 매핑 → COVERS 정렬 → 로드맵 | 10~14 |
| **maturity-diagnosis** | 우리 수준은? | Ownership×Capability 매트릭스 → 현재 수준 → 목표 수준 → 갭 → 액션 | 10~14 |
| **rightsize-plan** | 뭘 줄이는가? | 유휴 탐지 → 3 대안(유지/감축/변경) → 권고 → BEP/Risk | 12~16 |
| **commit-strategy** | 어떻게 약정? | 워크로드 분류(상시/변동/배치) → RI/SP/Spot 혼합 3 시나리오 → 비용 절감 | 10~14 |
| **review-runbook** | 어떻게 운영? | 주/월/분기 KPI → 게이트 6종 → 조치 흐름 → 책임자 R&R | 12~16 |
| **review-report** | 검증 결과는? | 4단계 정합성 → 게이트 통과 현황 → 미결 이슈 → 판정 | 10~12 |
| **maturity-transition** | 어떻게 성장? | 현재 → 6개월 → 12개월 단계별 Ownership×Capability 성장 경로 | 12~16 |

## 2. 도메인 특화 슬라이드 패턴 매핑

DMAP 가이드 패턴(A~F)에 FinOps 도메인 콘텐츠를 매핑:

| 콘텐츠 유형 | 권장 패턴 | 시각 요소 |
|------------|----------|----------|
| 3 가치 (절감/가속/신뢰) | E (2×3 또는 3열 카드) | 각 카드에 배지 + KPI 예시 |
| Ownership×Capability 매트릭스 | D (테이블+배지) 또는 B (이미지) | 축·좌표·셀 색상 표기 |
| KPI 대시보드 | F (멀티 섹션) | 4~6 KPI 카드 + 스파크라인 |
| 유휴 탐지 결과 | D (테이블) | 리소스·사용률·월 비용·절감 예상 |
| Right-sizing 3 대안 | E (3 카드) | 유지/감축/변경 — 각 장단점 |
| RI/SP/Spot 3 시나리오 | D (3열 표) + 배지 | Conservative/Base/Optimistic |
| BEP 곡선 | B + 이미지 | generate_image PNG + 우측 설명 |
| 게이트 6종 | C (플로우) 또는 E (카드) | 순서 흐름 또는 분류 카드 |
| FOCUS v1.3 스키마 | D (테이블) | 컬럼 정의·타입·예시 |
| AI 확장 컬럼 5종 | E (5 카드) | 각 컬럼 용도 + 예시 |
| 이상 탐지 5단계 | C (플로우) | 수집 → 분석 → 탐지 → 대응 → 예측 |
| 자동화 5단계 | C (플로우) | Crawl → Walk → Run 단계별 |
| 3 가치 × COVERS 매핑 | D (매트릭스 표) | 3×6 교차 표 |

## 3. 수치 표기 규칙 (MUST)

모든 절감액·예산·기간은 **범위 또는 3 시나리오**로 표기. 단일 숫자(거짓 정확도) 금지:

| ✅ 권장 | ❌ 금지 |
|--------|--------|
| `월 절감액: 2,400~3,600 USD (Base 3,000)` | `월 절감액: 3,000 USD` |
| `Phase 1: 3~6개월` | `Phase 1: 4개월` |
| `BEP: 8~14개월 (Conservative 14 / Base 11 / Optimistic 8)` | `BEP: 11개월` |

**예외**: 룰 ID, 게이트 통과 기준(예: CPU<40%), 실측값(예: 현재 MTTR 6시간)은 단일 숫자 허용.

## 4. 출처 표기 (MUST)

모든 수치·주장이 기재된 슬라이드 하단에 출처 캡션 추가:

```
출처: FOCUS v1.3 spec · 내부 CUR 2024-Q4 분석
```

- 스타일: 캡션 (Pretendard Regular 12pt · `#6B6B7B`)
- 위치: 슬라이드 최하단 푸터 위 (마스터 푸터 영역 침범 금지)
- 출처 없는 수치는 **[가정]** 배지(다크 배지 `#404155`)로 표시

## 5. 강조 배지 용어집 (FinOps 도메인)

DMAP 가이드의 강조 배지(녹색 `#059669`) 텍스트 표준:

| 배지 | 용도 |
|------|------|
| `WHY` | WHY 정의 섹션 표지 |
| `KPI` | KPI 대시보드·측정 지표 슬라이드 |
| `RI/SP` | 약정 전략 슬라이드 |
| `Spot` | Spot 인스턴스 전략 슬라이드 |
| `Gate` | 게이트 통과 조건 슬라이드 |
| `FOCUS` | FOCUS 스키마 슬라이드 |
| `BEP` | 손익분기 분석 슬라이드 |
| `Crawl/Walk/Run` | 자동화 성숙도 단계 |

다크 배지(`#404155`)는 카테고리명 — `Unit Economics`, `Anomaly Detection`, `Tag Policy`, `AI Cost` 등에 사용.

## 7. 교육용 일러스트 스타일 (image_generate 프롬프트 MUST)

FinOps 학습랩의 이미지는 다음 스타일 규칙을 따름 (Gemini 프롬프트 작성 시 MUST 포함):

- **배경**: 흰색(`#FFFFFF`) — 투명 배경 금지 (PPT 임베딩 시 경계 모호)
- **선**: 깔끔한 단선, 두께 2~3px, 검은색 또는 `#2C2926`
- **톤**: 전문적·미니멀 — 일러스트 내 텍스트 최소화 (텍스트는 PPT 본문에 배치)
- **컬러**: DMAP Accent 팔레트(`#4472C4`, `#ED7D31`, `#70AD47`, `#FFC000`)만 사용 (네온·형광 금지)
- **사이즈**: 1200×800 또는 1600×1000 (16:10 근접) — 슬라이드 우측 배치 시 여백 확보
- **캐릭터·로고·워터마크**: 금지 (저작권·브랜드 간섭 방지)
- **금지 요소**: AWS/Azure/GCP 공식 로고, 실제 인물 사진, 금액 표시 (숫자는 PPT 본문에서 관리)

## 8. 3 시나리오 컬러 규칙

Conservative / Base / Optimistic 3 시나리오를 시각화할 때 일관된 컬러 사용:

| 시나리오 | HEX | 용도 |
|---------|-----|------|
| Conservative | `#DC2626` | 보수적 가정 (빨강 계열) |
| Base | `#0D9488` | 기준 가정 (DMAP Sub 컬러와 유사) |
| Optimistic | `#059669` | 낙관적 가정 (녹색 계열) |

표·곡선·카드에서 이 컬러를 일관되게 적용 — 시청자가 시나리오를 색상만으로 식별 가능해야 함.

## 9. 최소/최대 장수 규칙

- **최소**: 10장 (표지·본문 흐름·결론 포함)
- **최대**: 20장 (부록 포함). 초과 시 섹션을 다른 산출물로 분할 권고.
- 학습용 발표는 **15~20분** 기준으로 설계 (슬라이드당 60~90초).
