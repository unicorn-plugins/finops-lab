# 서비스별 스케일링 정책 체크리스트

## 체크리스트 적용 기준

- 유휴 판정 기준: CPU 2주 평균 < 40%, Memory 2주 평균 < 60%  
- 자동화 실행은 본 체크리스트 범위 외 (별도 운영팀 승인 필요)  
- 스케일-인 임계값 도달 후 최소 72시간 모니터링 후 적용 권고

## AWS EC2 스케일링 정책

| 항목 | 내용 |
|------|------|
| 스케일-인 CPU 임계값 | 평균 < 40% (14일 연속) |
| 스케일-인 Memory 임계값 | 평균 < 60% (14일 연속) |
| 스케일-아웃 CPU 임계값 | 순간 > 80% (5분 연속) |
| 스케일-아웃 Memory 임계값 | 순간 > 85% (5분 연속) |
| 최소 인스턴스 수 | 2 (가용성 보장) |
| Downsize 전 사전 점검 | [ ] 피크 시간대 CPU/MEM 확인 [ ] 애플리케이션 팀 승인 [ ] 롤백 계획 수립 |
| Terminate 전 사전 점검 | [ ] 연결된 EBS 볼륨 정리 [ ] 보안그룹 정리 [ ] 스냅샷 백업 |
| 적용 후 모니터링 기간 | 최소 7일 |

| 항목 | 내용 |

| 적용 후 모니터링 기간 | 최소 7일 |

## AI 모델 스케일링 정책

| 항목 | 내용 |
|------|------|
| GPU 저활용 임계값 | 평균 < 60% (14일 연속) |
| 모델 다운그레이드 조건 | GPU < 60% + 품질 지표 허용 범위 내 |
| 다운그레이드 전 A/B 테스트 기간 | 최소 7일 |
| 품질 지표 측정 항목 | 응답 정확도, 응답 시간, 사용자 만족도 |
| 롤백 트리거 | 품질 지표 10% 이상 하락 시 즉시 롤백 |

## 공통 사전 점검 체크리스트

- [ ] 2주 이상 이용률 데이터 확보 확인  
- [ ] 해당 리소스 담당팀(오너) 승인 완료  
- [ ] 피크 시간대 예외 여부 확인 (배치 작업, 이벤트 기간 등)  
- [ ] 스냅샷/백업 최신 상태 확인  
- [ ] 롤백 절차 문서화 완료  
- [ ] 변경 후 알림 임계값 재설정 완료  
- [ ] ITSM 변경 요청(RFC) 등록 완료
